# QR スキャン機能 — 詳細仕様

このセッションで設計・確定した内容をまとめたもの。
コードや実装詳細は含まない。Claude Code 上での実装前の参照用。

---

## 1. QR スキーム定義

| 対象 | スキーム | 例 |
|------|----------|----|
| 服 | `dwg://g/{garment-id}` | `dwg://g/clx1a2b3c` |
| 収納場所 | `dwg://l/{location-id}` | `dwg://l/clx9z8y7x` |

- プレフィックス `dwg://` で服か場所かを即座に判別できる
- 同一カメラ画面で両方のスキャンを受け付ける（種別に応じて処理を分岐）
- ID は cuid2 推奨

---

## 2. 物理ラベル仕様

### 服側ラベル
- サイズ: 15mm × 15mm 以上
- 素材: 耐水ラベル用紙（A-ONE 31542 相当）
- 貼付場所: **服には直貼りしない**。保管用小袋の外側に貼付
- 理由: 繊細な生地を傷める可能性、洗濯時の剥がれ防止

### 場所側ラベル
- サイズ: 25mm × 25mm 以上（複数人が別角度からスキャンする想定）
- 貼付場所: 引き出し・ケースの**前面外側**
- 構成: **QR コード + NFC シールを両方貼付**
  - QR: iOS / Android 共通で読める
  - NFC: Android Chrome の Web NFC API 向け（より低摩擦なスキャン体験）

### iOS 制約への対応
- Web NFC API は **Android Chrome のみ**対応。iOS は未対応
- 対策: 同じ場所に QR と NFC を併設する。iOS ユーザーは QR を使う
- アプリ側での分岐不要。スキャン結果（ID）が同じであれば処理は共通

### 初期コスト目安
- 耐水ラベル用紙: 約 ¥700（200枚）
- NFC シール 20枚: 約 ¥400
- 合計: 約 ¥1,100

## 3. スキャンフロー

### 基本フロー（収納時）

```
1. スキャン画面を開く
2. 場所側 QR をスキャン
   → activeLocationId がセットされる
   → その場所の登録済みアイテムを信頼度付きで表示
   → 要確認アイテムがあれば機会確認ダイアログを表示（→ セクション 5）
3. 服側 QR を連続スキャン（何枚でも）
   → スキャンのたびにリストへ追加
4. 「この場所にしまう」ボタンでバッチ確定
   → 全アイテムの locationId = activeLocationId
   → 全アイテムの lastScannedAt = 現在時刻
   → status = 'stored'
```

### チェックアウトフロー（取り出し時）

```
1. スキャン画面を開く
2. 服側 QR をスキャン（場所 QR は不要）
3. 「取り出す」ボタンをタップ
   → locationId = null
   → status = 'checked_out'
   → checkedOutAt = 現在時刻
```

### スキャン順序のルール
- **場所 QR が先**、服 QR が後という順序を固定する
- 服 QR だけスキャンした場合はチェックアウトとして扱う
- 場所 QR の後に服 QR をスキャンした場合はチェックインとして扱う

## 4. スキャン画面の状態管理

スキャン画面は以下の状態を持つ。認証・グローバル状態管理の詳細は別途 Claude Code 上で設計する。

| 状態 | 説明 |
|------|------|
| `idle` | 初期状態。カメラ待機中 |
| `location_set` | 場所 QR スキャン済み。activeLocationId あり |
| `scanning` | 服 QR を連続スキャン中 |
| `confirming` | 機会確認ダイアログ表示中 |
| `committing` | API 送信中 |
| `done` | 完了。次のスキャンを促す |

## 5. 機会確認ダイアログ

### トリガー条件
場所 QR をスキャンしたとき、その場所に「要確認（信頼度 < 0.7）」のアイテムが 1 件以上ある場合に表示する。

### 表示内容
- その場所に登録されているアイテム一覧
- 各アイテムに信頼度バーと経過日数を表示
- 「全部ある ✓」ボタンと「ズレを直す」ボタン

### 「全部ある ✓」タップ時の処理
- その場所の全登録アイテムの `lastScannedAt` を現在時刻にリセット
- 信頼度が 1.0 に戻る
- ダイアログを閉じてスキャン継続

### 「ズレを直す」タップ時の処理
- アイテムごとに「ある / ない」を選択できるモードへ移行
- 「ない」を選んだアイテムは `status = 'unknown'` に更新

### 設計意図
引き出しを開けた状態 = ユーザーがその場所の前に物理的にいる瞬間。
このタイミングで確認を差し込むことで、確認コストをほぼゼロにする。

## 6. 信頼度モデル

### 概念
「このシステムが、その服が登録済みの場所にある、と確信できる度合い」を 0.0〜1.0 で表す。

### 計算方法
- `lastScannedAt`（最後に QR スキャンされた時刻）からの経過日数で線形に減衰
- `confidenceDecayDays`（デフォルト 30）で 0 に到達する
- DB には保存しない。常に計算で導出する

### 減衰パラメータの調整
服ごとに `confidenceDecayDays` を設定できる:
- よく着る服: 14 日
- 通常: 30 日（デフォルト）
- 季節物: 90 日

### 信頼度ゾーン

| ゾーン | 範囲 | 表示 | 意味 |
|--------|------|------|------|
| 確定 | 0.70〜1.00 | 緑 | スキャンから日が浅い。信頼できる |
| 要確認 | 0.30〜0.69 | 黄 | しばらく経過。次の機会に確認を |
| 不明 | 0.00〜0.29 | 赤 | 長期間スキャンなし。週次通知対象 |

### 信頼度リセットのタイミング
- チェックインスキャン時（`lastScannedAt` 更新）
- 機会確認の「全部ある」タップ時
- 孤立チェックアウト解決の「しまった」選択後のスキャン時

## 7. 孤立チェックアウト検出

### 定義
`status = 'checked_out'` かつ `checkedOutAt` から **3 日以上**経過したアイテム。

### 検出タイミング
- アプリ起動時にクライアント側でチェック
- 週次ダイジェスト生成時にサーバー側でチェック

### 解決 UI（3 択）
どの選択肢を選んでも必ずシステムの状態が前進する設計にする。

| 選択肢 | 処理 | 意図 |
|--------|------|------|
| しまった | 場所 QR スキャン画面へ誘導 → チェックイン処理 | 正常収納 |
| まだ使用中 | アラートを 7 日延期 | 積極的な使用中を記録 |
| なくした | `status = 'lost'` に更新 | 紛失として記録 |

## 8. 週次ダイジェスト

### 実行タイミング
毎週月曜 9:00 UTC（Cron Trigger）

### 通知対象
- 信頼度 0〜0.29（不明ゾーン）のアイテム
- 孤立チェックアウト（3 日以上未収納）のアイテム

### 通知設計の方針
- **個別アイテムごとの通知は出さない**（通知疲れ防止）
- 週 1 回にまとめて「確認が必要な服 N 点」として通知
- 「次に引き出しを開けたときに確認すればいい」という文面にする
- 急かさない。プレッシャーを与えない。

### Cloudflare 構成
```
Cron Trigger（毎週月曜）
  → Workers でダイジェスト内容を生成
  → Queues に投入
  → Queues コンシューマーが通知を送信
```

## 9. API エンドポイント一覧（QR 関連）

| Method | Path | 説明 |
|--------|------|------|
| POST | `/api/scan/checkin` | チェックイン（場所 + 服のリスト） |
| POST | `/api/scan/checkout` | チェックアウト（服 ID） |
| POST | `/api/scan/confirm-all` | 機会確認（場所の全件 lastScannedAt リセット） |
| POST | `/api/scan/confirm-partial` | 機会確認（個別に ある/ない を送信） |
| POST | `/api/scan/orphan-resolve` | 孤立チェックアウト解決（3 択） |

## 10. QR ラベル生成機能

### 用途
初期セットアップ時に服・収納場所の QR ラベルを印刷する。

### 出力形式
- PDF（pdfkit）で一括出力
- A4 用紙に並べて印刷できるレイアウト

### 生成対象
- 服 QR: `dwg://g/{id}` + 服の名前を小テキストで添える
- 場所 QR: `dwg://l/{id}` + 場所のラベル名を添える

### 運用
- 新しい服を登録するたびに個別に印刷できる画面も用意する
- ブラウザの印刷ダイアログを使う（アプリ内プレビュー付き）
